Framework news

New features and functions.



CHANGELOG

Live modules, models, controllers and views

This is the first best feature in the latest version. You can install some module / model / controller and view from own repository or from total.js repository. Framework supports install and uninstall features.

Framework supports middleware

This is the second best feature in the latest version. Framework supports similar functions like express.js for using of middleware (example: passport.js). Framework knows 4 types of middleware: global middleware, for the web routing, for the websocket routing and for the file routing.

New start scripts

From now we can run the framework easily. Example for understanding:
require('total.js').http('debug') or require('total.js').http('release') or require('total.js').http('test').

Image auto-resizer

You can define relative URL address and framework automatically resizes all images according your needs. You can set some additional properties: grayscale, sepia, blur, flip, flop and rotate. This feature has great functionality. Requirements: ImageMagick or GraphicsMagick. Here is an example.

LESS CSS is replaced to JS CSS

Framework keeps Auto Vendor Prefixer and adds JavaScript for generating CSS. How does it work? Framework finds all comments in CSS which contain these keywords: var or function. Then is the code compiled and evaluated. Picture below describes this feature. Here is an example.

View engine supports: looping

View engine was extended about the foreach feature for rendering arrays. So you can render some Array into the current view or layout without using templates.

View engine supports: inline helpers

Framework supports inline definition of helpers in views. The idea for this feature is taken from ASP.MVC RAZOR engine and is very helpful.

View engine supports: sections

View engine supports: assign value

Finally view engine supports: assign value to the object. These properties are supported in views: repository, model, global, session, user, get, post and config.

View engine supports: nested condition

New config file for both modes (debug and release)

New config file is "config" and it's for both modes (debug and release). Other configs have higher priority and can rewrite this config. All config files are optional. Here is an example.

Send e-mail directly from the controller

New version supports send e-mail from the controllor. Body of e-mail is rendered view. New version supports send e-mail from the controller. Body of e-mail is rendered from an view. Here is an example.

New test functions

Framework has a new test functions for testing applications. New functions support testing routes and testing business logic. Test request supports flags (same than classic route flags) and cookies. Here is an example.
