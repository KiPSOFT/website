Framework news

New features and functions.


Picture auto-resizer

This is the best feature in latest version. You can define relative URL address and framework automatically resize all pictures according your needs. You can set some additional properties: grayscale, sepia, blur, flip, flop and rotate. This feature has great functionality. Requirements: ImageMagick or GraphicsMagick.

LESS CSS is replaced to JS CSS

In new version you can use JavaScript for generating CSS. How does it work? Framework finds all comments in CSS which contain these keywords: var or function. Then is code compiled and evaluated. Picture below describes this feature:

View engine supports: foreach

View engine was extended about the foreach feature for rendering arrays. So you can render some Array into the current view or layout without using templates.

View engine supports: assign value

Finally view engine supports assign value to object. Are supported these properties in views: repository, model, global, session, user, get, post and config.

Older template engine was removed

Older template engine was removed and commented code for repeating is currently obsolete. Picture below describes new functionality of templates.

New config file for both modes (debug and release)

New config file is "config" and it's for both modes (debug and release). Other configs have higher priority and can rewrite this config. All config files are optional.

New test functions

Framework has a new test functions for testing applications. New functions support testing routes and testing business logic. Test request supports flags (same than classic route flags) and supports cookies.

Total.js package manager

This is a new binary/executable file for Terminal/CMD. With the help of TPM you can install/uninstall any total.js module or templates. You can create own package repository on own server. NPM has only modules and TPM (Total.js Package Manager) will contain e.g. whole projects. TPM will not replace NPM.
