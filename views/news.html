Framework news

New features and functions.


Image auto-resizer

This is the best feature in the latest version. You can define relative URL address and framework automatically resizes all images according your needs. You can set some additional properties: grayscale, sepia, blur, flip, flop and rotate. This feature has great functionality. Requirements: ImageMagick or GraphicsMagick.

LESS CSS is replaced to JS CSS

Framework keeps Auto Vendor Prefixer and adds JavaScript for generating CSS. How does it work? Framework finds all comments in CSS which contain these keywords: var or function. Then is the code compiled and evaluated. Picture below describes this feature:

View engine supports: looping

View engine was extended about the foreach feature for rendering arrays. So you can render some Array into the current view or layout without using templates.

View engine supports: inline helpers

Framework supports inline definition of helpers in views. The idea for this feature is taken from ASP.MVC RAZOR engine and is very helpful.

View engine supports: assign value

Finally view engine supports: assign value to the object. These properties are supported in views: repository, model, global, session, user, get, post and config.

View engine supports: nested condition

Older template engine was removed

Older template engine was removed and commented code for repeating is currently obsolete. Picture below describes new functionality of templates.

New config file for both modes (debug and release)

New config file is "config" and it's for both modes (debug and release). Other configs have higher priority and can rewrite this config. All config files are optional.

New test functions

Framework has a new test functions for testing applications. New functions support testing routes and testing business logic. Test request supports flags (same than classic route flags) and cookies.
